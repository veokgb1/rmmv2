# 功能键设计（RMM V2）

## F1 快捷记账（主入口）

### 目标
快速录入一条或多条记账信息（文字为主）

---

### 阶段结构

#### 阶段1：输入
- 支持文字输入
- 预留语音入口（占位）
- 预留图片入口（占位）

---

#### 阶段2：候选生成
- 本地解析输入内容
- 生成候选记录（不入库）
- 展示为只读结构

---

#### 阶段3：候选确认
- 提供“确认记录”按钮
- 点击后：
  - 标记 candidateConfirmed = true
  - 提示：已确认（未入库）
- 禁止重复确认

---

#### 阶段4：状态收口（当前阶段）
- 新候选生成 → 重置确认状态
- 弹层关闭 → 清空状态
- UI状态必须与数据状态一致

---

### 状态模型

- currentCandidate
- candidateConfirmed

---

### 当前限制（必须遵守）

- 不允许编辑字段
- 不允许写入数据库
- 不允许网络请求
- 不接 AI / OCR / 语音
