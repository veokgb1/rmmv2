# RMM V2 项目状态与记忆 (Auto Memory)

## 1. 系统架构目标 (多层账本系统)
- **Formal:** 正式账本
- **Temp:** 临时记录 (承接不确定记录)
- **Pending_link:** 待匹配凭证 (图片先入池，再绑定)
- **Difficulty:** 复杂处理层

## 2. 核心模块定义 (九宫格)
- F1 快捷记账：最快生成一条“可确认但未入库”的记录。
- F2 临时录入：手动录入，可编辑/删除，可转正。
- F3 精准绑定/解绑：处理“记录 ↔ 凭证”的一对一/一对多关系。
- F4 待匹配池：集中处理未匹配的图片/凭证。
- F5 全局排查中心：搜索账目、图片及绑定关系。
- F6 难度处理中心：处理多对多的复杂冲突。
