# RMM V2 开发系统指令 (Harness Rules)

## 1. 绝对红线 (Red Lines)
- **纯前端沙箱：** 当前系统处于 UI 与本地逻辑构建阶段。严禁编写任何将数据直接写入 Firebase (formal/temp) 的代码。
- **中文与编码保护：** 所有的代码注释、UI 界面文案必须保持原样的简体中文，严格使用 UTF-8 编码，绝不允许输出乱码。
- **向后兼容：** 绝不修改或删除旧有核心字段（`month`, `date`, `voucherIds` 等），必须保持对旧版 `fetchLedger` 的兼容。

## 2. 交互纪律 (Interaction Protocol)
- **强制先决条件：** 在执行任何开发任务前，必须以项目根目录下的 `MEMORY.md` 作为唯一的事实来源。
- **Plan-First 模式：** 接收到修改需求后，不准直接输出代码。必须先输出【执行计划】（说明要修改的函数、具体逻辑、预期状态）。
- **等待授权：** 输出计划后，必须显式提问：“DUKA，请确认是否按此计划执行？” 得到同意后才可编写代码。
